## 設計成果物の英語名

システムの設計成果物は、英語では文脈やニュアンスによっていくつかの表現が使い分けられます。最も一般的で汎用的なのは **Deliverables** です。

用途に合わせた主な3つの表現を紹介します。

### 1. Deliverables（デリバラブルズ）
**最も一般的で、ビジネスやプロジェクト管理の文脈**で使われます。
「（クライアントや後工程へ）納品すべきもの」というニュアンスが強く、契約やマイルストーンに関連して語られる際に使われます。[[1](https://www.google.com/url?sa=E&q=https%3A%2F%2Fvertexaisearch.cloud.google.com%2Fgrounding-api-redirect%2FAUZIYQH4SqRR7_w-4TmvcATuXEoN1O8cOZ5PcdG5CzV44mboTNOYo7bjRJQ2RxP4QQ9AcM-mGX5O4nce69kkSrWFdyheVzTdUgVXGJNjeYUHO8BHPp0iLzVsFpliLNvp84_v4RJ0rPX35wjRIE1ETQMgJX91vHs%3D)]

*   **System Design Deliverables**: システム設計の成果物
*   **Project Deliverables**: プロジェクト全体の成果物[[2](https://www.google.com/url?sa=E&q=https%3A%2F%2Fvertexaisearch.cloud.google.com%2Fgrounding-api-redirect%2FAUZIYQFAqpmLcXXyzM6fN5c2Fj0L6PeVDrP9EbFBbN41EQgxwHaVLmUeRWYNexR-MT72OhSlGzbj-YWFyM2ej1wEo4-qOxYAF2Gozmq-WazbMfOAkbKHkZVfoUJFxE5UXKTxxO5-vE23HuQ%3D)]

### 3. Design Documents / Specifications (Specs)
**より具体的で、書類や定義書そのもの**を指す場合に日常的に使われます。[[2](https://www.google.com/url?sa=E&q=https%3A%2F%2Fvertexaisearch.cloud.google.com%2Fgrounding-api-redirect%2FAUZIYQFAqpmLcXXyzM6fN5c2Fj0L6PeVDrP9EbFBbN41EQgxwHaVLmUeRWYNexR-MT72OhSlGzbj-YWFyM2ej1wEo4-qOxYAF2Gozmq-WazbMfOAkbKHkZVfoUJFxE5UXKTxxO5-vE23HuQ%3D)]
「設計書」や「仕様書」と呼びたい場合はこちらが最も自然です。

*   **Design Documents**: 設計書（略して **Design Docs**）
*   **Design Specifications**: 設計仕様書（略して **Design Specs**）

---

### その他の関連表現
*   **Work Products**: 作業の結果として生み出されたもの全般を指す、やや堅い表現です。
*   **Outputs**: 単に「出力されたもの」という一般的な意味で使われます。
